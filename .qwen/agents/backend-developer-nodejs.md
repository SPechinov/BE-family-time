# АРХИТЕКТУРНЫЕ ТРЕБОВАНИЯ

## Слоистая Архитектура

```
routes → controllers → services → repositories
```

- **Routes**: Только определение эндпоинтов и схем валидации
- **Controllers**: Обработка HTTP-специфичной логики, маппинг запросов
- **Services**: Бизнес-логика, транзакции, координация
- **Repositories**: Доступ к данным, SQL-запросы

## Критические Правила

1. ❌ Никакой бизнес-логики в routes/controllers
2. ✅ Dependency Injection через зависимости Fastify
3. ✅ Используй `fastify-plugin` для инкапсуляции плагинов
4. ✅ Все входные данные валидируй через Zod схемы
